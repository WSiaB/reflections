How is the staging area different from the working directory and the repository? What value do you think it offers?

	The staging area are the files ready to be commit. The working directory contains both tracked and untracked files. This allows more granular control of what is in the repository and smaller commits.

How can you use the staging area to make sure you have one commit per logical change?

	It is an intermediate step that allows you to designate only the changed files related to the commit.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

	For experimenting, testing new features.

How do the diagrams help you visualize the branch structure?

	They allow you to trace the lineage of your branches.
